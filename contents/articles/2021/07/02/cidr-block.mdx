### 用途

セキュリティーグループや AWS におけるネットワークの構築全般に使われる

AWS のサービスが使用できる IP の範囲を指定するときに使う

####　例
セキュリティーグループで言うと。。。
172.32.0.0/32
-> 172.32.0.0 一つしか許可しない

0.0.0.0/0
-> 全ての IP アドレスを許可

192.168.0.0/26
-> 31 - 26 = 5
-> 2 \*\* 5 = 64
192.168.0.0 - 192.168.0.63 (合計 64 の IP が許可される)

###　構成
192.168.0.0 <- IP
/24 <- サブネットマスクと呼ばれ CIDR の特徴
