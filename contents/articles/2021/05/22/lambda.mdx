### AWS Lambda とは？

一般的に Serverless と呼ばれています。名前どおりで一からサーバーを立てなくても AWS 側がサーバー（のようなもの）を用意してくれます。
ただし、ここで注意しないといけないのは、全くサーバーを必要としないわけでなく、あくまで開発している私たちがサーバーの立ち上げをいちいちしなくても良いと言うことです。(AWS 側がサーバーの立ち上げをしてくれると言うこと。)

### メリット

Lambda を使うメリットは数多くありますがその中でも特に便利と感じるのは以下の２点

1 時間短縮ができる（サーバーを一から立ち上げる時間、サーバーについての学習を減らせる）

2 本来のサーバーであれば常にどこかのポートに依存し、常にそのポートから HTTP リクエストを聞き続けていなければならない。しかし Lambda の場合、
ユーザー側から連絡が来た時だけ（これを Event と言う）、サーバーを起動し対処することができる。
->消費電力が削減できるので使用コストも本来に比べ、大幅に削減できる

### 立ち上げ方

どの AWS のサービスを利用するにも共通して主に３つ方法が存在します
１　 AWS のコンソールを使う
これは実際に AWS のホームページから立ち上げを行っていく方法です
→ 　しかし、インフラが複雑になればなるほどコンソールからの操作は全体を把握しづらくなるので長期的に見るとおすすめできない

２　 CLI を使う
AWS CLI をターミナルから使っている方法です。コマンドさえ覚えていればコンソールよりも時間は短縮できますが１と同様、インフラを立ち上げるために使うと言うよりも、小さい変更をくわえたい時に主に利用していきます。
CI/CD に組み込んだりすることも多々あります。

３ Infrastructure as a Service 系統のツールを使っていく
-> Terraform や Cloudformation などのようにあらかじめどのようにインフラを整えていくかの指示書をファイルなどに記載して、実行していくアプローチ。
長期的にみた時には修正などもしやすくおすすめです。
